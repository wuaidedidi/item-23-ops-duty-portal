# 任务计划制定

将确认的设计或用户需求拆解为具体可执行的工程任务。

## 步骤

1. **扫描项目结构**：用 `find_by_name` 和 `code_search` 了解现有代码结构和技术栈。

2. **拆解任务**：将工作拆分为 2-5 分钟的小任务，用 `todo_list` 工具创建清单。每个任务必须包含：
   - 要修改/创建的 **具体文件路径**
   - 要做的 **具体变更**（不是模糊描述）
   - **验证方法**（如何确认这个任务完成了）

3. **排序依赖**：按依赖关系排序任务，被依赖的任务排在前面：
   - 数据层/模型 → 业务逻辑 → UI层 → 集成测试

4. **展示计划**：将完整任务清单展示给用户确认。格式：
   ```
   任务 1: [文件] 做什么
   任务 2: [文件] 做什么
   ...
   ```

5. **用户确认后开始执行**：确认后自动进入 `/execute` 工作流。

## 任务拆分原则

- 每个任务只改 1-2 个文件
- 每个任务改动不超过 50 行代码
- 每个任务完成后项目仍可运行（不破坏现有功能）
- 任务描述具体到函数名和行为，不用模糊词语
