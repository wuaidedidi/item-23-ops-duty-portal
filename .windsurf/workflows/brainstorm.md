# 需求脑暴与设计

在动手写任何代码之前，先完成需求理解和方案设计。

## 步骤

1. **理解需求**：向用户提问，明确以下内容：
   - 要解决什么问题？
   - 目标用户是谁？
   - 核心功能有哪些？
   - 有哪些约束条件（技术栈、性能、兼容性）？

2. **探索方案**：提出 2-3 种实现方案，对比优劣：
   - 方案 A：描述 + 优点 + 缺点
   - 方案 B：描述 + 优点 + 缺点
   - 推荐方案及理由

3. **输出设计文档**：将确认后的设计分段展示给用户验证，每段不超过 10 行：
   - 数据模型设计
   - 模块划分与职责
   - 接口/事件定义
   - UI 交互流程

4. **用户确认**：每段设计都需要用户确认后才进入下一段。全部确认后，输出完整设计摘要。

5. **进入计划阶段**：设计确认后，自动触发 `/plan` 工作流，将设计拆解为可执行的任务计划。

## 注意事项

- 不要在脑暴阶段写任何实际代码
- 保持设计文档简洁，避免过度设计
- 如果用户已经有清晰的需求描述，可以跳过提问直接进入方案探索
