# 任务执行与审查

按照 `/plan` 制定的任务清单，逐个执行并审查。

## 执行循环

对 `todo_list` 中每个 `pending` 任务，重复以下步骤：

1. **标记开始**：将任务标记为 `in_progress`

2. **阅读上下文**：用 `read_file` 完整阅读相关文件，理解现有代码

3. **实现变更**：用 `edit` 或 `multi_edit` 进行最小化修改
   - 优先修改上游根因，不打下游补丁
   - 保持现有代码风格
   - 不改动无关代码
   - 不删除现有注释

4. **自审检查**：实现后立即检查：
   - 修改是否完整实现了任务描述的功能？
   - 是否引入了新的问题？
   - 导入/依赖是否正确？
   - 事件名、ID、接口是否与其他模块匹配？

5. **标记完成**：自审通过后标记为 `completed`

6. **继续下一个任务**

## 全部完成后

7. **整体验证**：打开浏览器验证所有改动：
   - 逐个测试已实现的功能
   - 检查控制台无报错
   - 检查 UI 显示和交互正常

8. **发现新问题**：如果验证中发现问题，添加到 `todo_list` 并继续执行循环

9. **输出报告**：全部通过后，输出完成摘要：
   - 完成了哪些任务
   - 修改了哪些文件
   - 验证结果

## 原则

- 一次只做一个任务，不跳跃
- 每个任务完成后项目必须可运行
- 遇到阻塞问题立即告知用户，不猜测
